OpenShift-Kubernetesдёӯжҳ“зҗҶи§Јй”ҷиҜҜзҡ„й—®йўҳжұҮжҖ»

еҸ‘иЎЁдәҺ2020-05-20|жӣҙж–°дәҺ2025-03-28
|йҳ…иҜ»йҮҸ:

###1. readinessеҒҘеә·жЈҖжҹҘдёҖзӣҙеӨұиҙҘпјҢPodдјҡдёҚдјҡйҮҚеҗҜпјҹ

дёҚдјҡпјҢдёҖзӣҙеӨ„дәҺRunningзҠ¶жҖҒпјҢдҪҶNot ReadyгҖӮ

2. PodйҮҚеҗҜж—¶пјҢPodзҡ„е®№еҷЁжҳҜйҮҚеҗҜпјҢиҝҳжҳҜйҮҚж–°еҲӣе»әпјҹ

PODе®№еҷЁдёҚеҠЁпјҢеә”з”Ёе®№еҷЁжҳҜйҮҚж–°еҲӣе»әгҖӮPOD IPдҝқжҢҒдёҚеҸҳпјҢдҪҶеә”з”Ёе®№еҷЁдёӯзҡ„дёҙж—¶ж–Үд»¶дјҡдёўеӨұгҖӮ

###3. еҰӮжһңжӣҙж”№Deploymentзҡ„й…ҚзҪ®еҗҺпјҢй©¬дёҠе°Ҷй…ҚзҪ®жҒўеӨҚеӣһеҺ»пјҢеә”з”ЁPodжҳҜеҗҰдјҡи§ҰеҸ‘йҮҚж–°йғЁзҪІпјҹ
дёҚдјҡпјҢжҒўеӨҚеӣһеҺ»еҗҺдёҚдјҡеҲӣе»әж–°зҡ„RSгҖӮ

4. master_cluster_hostnameдёҺmaster_cluster_public_hostnameзҡ„еҢәеҲ«

master_cluster_hostnameжҳҜйӣҶзҫӨеҶ…йғЁз»„д»¶дҪҝз”Ёзҡ„еҹҹеҗҚ
master_cluster_public_hostnameжҳҜйӣҶзҫӨеӨ–йғЁдҪҝз”Ёзҡ„еҹҹеҗҚ
###5. еҜ№дәҺй«ҳеҸҜз”ЁйӣҶзҫӨдёӯпјҢ3дёӘmasterиҠӮзӮ№жҳҜе®Ңе…ЁзӣёеҗҢзҡ„еҗ—пјҹopenshift

дёҚжҳҜпјҢиҷҪз„¶е®ғд»¬зҡ„иҝҗиЎҢжҖҒжҳҜе®Ңе…ЁдёҖиҮҙзҡ„пјҢдҪҶжҳҜеңЁOpenShiftзҡ„иҝҗз»ҙи„ҡжң¬дёӯжҳҜжңүеҢәеҲҶзҡ„пјҢеҰӮйӣҶзҫӨжү©е®№ж—¶гҖӮжңүдёҖеҸ°жҳҜдё»masterпјҲinventoryдёӯзҡ„з¬¬дёҖеҸ°пјүпјҢе®ғдјҡеҚ•зӢ¬дҝқеӯҳйӣҶзҫӨзҡ„иҜҒд№ҰеҸҠдёҖдәӣй…ҚзҪ®гҖӮ

###6. еҜ№дәҺstatefulsetзұ»еһӢзҡ„е·ҘдҪңиҙҹиҪҪпјҢеҸӘиғҪйҖҗдёӘеҗҜеҠЁеҗ—пјҹ

е№¶дёҚжҳҜпјҢstatefulsetзұ»еһӢзҡ„workloadй»ҳи®ӨзЎ®е®һжҳҜйҖҗдёӘеҗҜеҠЁпјҢдҪҶжҳҜеҸҜд»ҘйҖҡиҝҮжӣҙж”№е®ғзҡ„еҗҜеҠЁзӯ–з•ҘжқҘе®һзҺ°е№¶иЎҢеҗҜеҠЁгҖӮ
spec.podManagementPolicyеҖјй»ҳи®ӨдёәOrderedReadyпјҢеҰӮжһңи®ҫзҪ®дёәParallelпјҢе°Ҷдјҡе№¶иЎҢеҗҜеҠЁе’ҢеҲ йҷӨPodгҖӮ

###7. statefulsetзұ»еһӢзҡ„е·ҘдҪңиҙҹиҪҪзҡ„еӨҡдёӘеүҜжң¬пјҢиғҪе…ұз”ЁдёҖеқ—е…ұдә«еӯҳеӮЁеҗ—пјҹ

еҪ“з„¶еҸҜд»ҘпјҢеңЁstatefulsetзҡ„зј–жҺ’дёӯдёҚдҪҝз”ЁvolumeTemplateпјҢиҖҢжҳҜдёҺdeploymentиө„жәҗдёҖж ·и®ҫзҪ®е…ұдә«еӯҳеӮЁе°ұеҸҜд»ҘдәҶгҖӮ

###8. еҰӮжһңеә”з”Ёеӯҳжҙ»жЈҖжҹҘ/е°ұз»ӘжЈҖжҹҘжңӘйҖҡиҝҮзҡ„жғ…еҶөдёӢпјҢжҳҜеҗҰиғҪйЎәеҲ©ең°иҝӣиЎҢеә”з”Ёжӣҙж–°пјҹ

дёҚдёҖе®ҡпјҢиҝҷдёӘеҸ–еҶідәҺеә”з”Ёзҡ„еҚҮзә§зӯ–з•Ҙпјҡж»ҡеҠЁеҚҮзә§пјҢmaxSurgeдёҺmaxUnavailableзҡ„й…ҚзҪ®гҖӮеҰӮжһңеә”з”ЁжҳҜдёҖдёӘеүҜжң¬пјҢmaxSurgeдёҺmaxUnavailableзҡ„еҖјдёәй»ҳи®ӨеҖј25%пјҢе°Ҷж— жі•е…ій—ӯж—§зүҲжң¬еә”з”ЁгҖӮ

  • maxSurgeпјҡеҚҮзә§иҝҮзЁӢдёӯжңҖеӨҡеҸҜд»ҘжҜ”еҺҹе…Ҳи®ҫзҪ®еӨҡеҮәзҡ„PODж•°йҮҸгҖӮеҸҜд»ҘжҳҜе…·дҪ“дёӘж•°пјҢд№ҹеҸҜд»ҘзҷҫеҲҶжҜ”пјҲиҝӣдёҖжі•еҸ–ж•ҙпјү
  • maxUnavailableпјҡеҚҮзә§иҝҮзЁӢдёӯжңҖеӨҡжңүеӨҡе°‘дёӘPODеӨ„дәҺж— жі•жҸҗдҫӣжңҚеҠЎзҡ„зҠ¶жҖҒгҖӮеҸҜд»ҘжҳҜе…·дҪ“дёӘж•°пјҢд№ҹеҸҜд»ҘзҷҫеҲҶжҜ”пјҲеҺ»е°ҫжі•еҸ–ж•ҙпјү

###9. еҰӮжһңеә”з”ЁеҒҡдәҶеӨҡеүҜжң¬пјҢиҖҢдё”йғЁзҪІеңЁдёҚеҗҢзҡ„иҠӮзӮ№дёҠпјҢеҚідҪҝе®ҝдё»жңәеҸ‘з”ҹе®•жңәпјҢеҜ№дёҡеҠЎд№ҹдёҚдјҡжңүеҪұе“ҚпјҢжӯЈзЎ®еҗ—пјҹ

иҝҷдёӘз»“и®әе№¶дёҚжӯЈзЎ®гҖӮеӣ дёәдҪңдёәеҶ…йғЁиҙҹиҪҪеқҮиЎЎзҡ„Serviceе№¶дёҚиғҪй©¬дёҠж„ҹзҹҘеә”з”Ёзҡ„й—®йўҳпјҢжүҖд»ҘдёҚдјҡй©¬дёҠдё»еҠЁеү”йҷӨж•…йҡңеә”з”ЁпјҢеҗҢж—¶ServiceдёҖиҲ¬дҪҝз”ЁпјҲocpпјүiptablesдҪңдёәд»ЈзҗҶж–№ејҸпјҢе®ғжІЎжңүдҪҝз”ЁеӨұиҙҘеҲҷдҪҝз”ЁеҲ«зҡ„иҠӮзӮ№зҡ„жңәеҲ¶пјҢжүҖд»ҘеңЁдёҖж®өж—¶й—ҙеҶ…иҜ·жұӮдјҡиў«еҸ‘йҖҒеҲ°й—®йўҳиҠӮзӮ№гҖӮ

й»ҳи®Өжғ…еҶөеҰӮдёӢпјҡ

  1. 40sжЈҖжөӢеҗҺпјҢжөҒйҮҸдјҡд»ҺиҙҹиҪҪеқҮиЎЎдёӯеү”йҷӨгҖӮ
  2. 5еҲҶй’ҹд№ӢеҶ…е®ҢжҲҗйҮҚеҗҜпјҢpodnameдёҚеҸҳпјҢpodзҡ„ipдјҡжӣҙж–°гҖӮ
  3. еҰӮжһң5еҲҶй’ҹеҗҺnodeиҠӮзӮ№д»Қз„¶жІЎжңүжҒўеӨҚпјҢйӣҶзҫӨдјҡе°Ҷй—®йўҳpodйҮҚж–°и°ғеәҰгҖӮ

###10. NFSдҪңдёәжҢҒд№…еҢ–еҚ·жҢӮиҪҪеңЁеә”з”ЁдёӯпјҢеҰӮжһңNFSжңҚеҠЎж•…йҡңеҗҺжҒўеӨҚпјҢе®№еҷЁжҳҜеҗҰдјҡиҮӘеҠЁйҮҚж–°жҢӮиҪҪеӯҳеӮЁжңҚеҠЎпјҹ
жҳҜзҡ„гҖӮ

11. еңЁDockerзҡ„жңҚеҠЎеҷЁдёҠпјҢе…ій—ӯ/йҮҚеҗҜiptablesпјҢдёҠйқўзҡ„е®№еҷЁзҪ‘з»ңдҫқз„¶жӯЈеёёеҗ—пјҹ

ејӮеёёгҖӮdockerе®үиЈ…е®ҢжҲҗеҗҺпјҢдјҡиҮӘеҠЁжҺҘз®ЎiptablesжҲ–иҖ…firewalldпјҢеңЁdocker runзҡ„ж—¶еҖҷпјҢдјҡиҮӘеҠЁеҫҖiptablesйҮҢеҠ е…Ҙи§„еҲҷпјӣжүҖд»ҘеҪ“iptablesйҮҚеҗҜеҗҺдјҡдёўеӨұпјҢеҸӘжңүеҶҚйҮҚеҗҜdockerе°ұеҘҪдәҶзҡ„еҺҹеӣ гҖӮ
еңЁдҪҝз”ЁSystemd ж—¶пјҢ firewalld дјҡеңЁ Docker д№ӢеүҚеҗҜеҠЁпјҢдҪҶжҳҜеҰӮжһңдҪ еңЁ Docker еҗҜеҠЁд№ӢеҗҺеҶҚеҗҜеҠЁжҲ–иҖ…йҮҚеҗҜ firewalld пјҢе°ұйңҖиҰҒйҮҚеҗҜ Docker иҝӣзЁӢдәҶгҖӮ
